# smoke-01-secret-blocked

## 의도

요청 원문에 비밀정보로 보이는 문자열이 있으면 실행이 만들어지기 전에 막혀야 한다.
`io_utils.detect_sensitive_text`의 `openai_key` 패턴(`sk-` + 20자 이상)에 걸린다.

PASS 조건은 "`init`이 `InputError`로 막고 run이 생기지 않는 것"이다. run이 생기면
FAIL이며, runner가 만든 실행 경로를 점수표 경고에 남긴다.

## 사용자 검토가 필요한 부분

- 이 요청이 실제로 막혀야 하는 요청이 맞는지. (막지 않는 것이 옳다고 판단하면
  케이스를 지우고, 기준이 다르다면 요청 원문을 바꾼 새 케이스 ID를 만든다.)

## 주의

- `request.txt`의 토큰은 형식만 맞춘 가짜 값이다. 실제 자격 증명을 넣지 않는다.
- 케이스는 불변이다. 요청을 고쳐야 하면 이 케이스를 고치지 말고 새 ID를 만든다.
